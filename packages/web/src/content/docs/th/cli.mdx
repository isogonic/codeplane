CodePlane CLI จะเริ่มต้น [TUI](/docs/tui) เป็นค่าเริ่มต้นเมื่อเรียกใช้โดยไม่มีอาร์กิวเมนต์

```bash
codeplane
```

นอกจากนี้ยังรับคำสั่งและแฟล็ก ซึ่งช่วยให้คุณใช้ CodePlane ผ่านโปรแกรมได้

```bash
codeplane run "Explain how closures work in JavaScript"
```

---

### tui

เริ่ม TUI เพื่อใช้ CodePlane แบบโต้ตอบ

```bash
codeplane [project]
```

#### แฟล็ก

| แฟล็ก        | สั้น | คำอธิบาย                                                   |
| ------------ | ---- | ---------------------------------------------------------- |
| `--continue` | `-c` | ดำเนินการต่อจากเซสชันล่าสุด                                |
| `--session`  | `-s` | ID เซสชันเพื่อดำเนินการต่อ                                 |
| `--fork`     |      | แยกเซสชันเมื่อทำต่อ (ใช้กับ `--continue` หรือ `--session`) |
| `--prompt`   |      | พรอมต์เริ่มต้นที่จะใช้                                     |
| `--model`    | `-m` | โมเดลที่จะใช้ในรูปแบบ provider/model                       |
| `--agent`    |      | เอเจนต์ที่จะใช้                                            |
| `--port`     |      | พอร์ตที่จะฟัง                                              |
| `--hostname` |      | ชื่อโฮสต์ที่จะฟัง                                          |

## คำสั่ง

CodePlane CLI ยังมีคำสั่งย่อยอีกมากมาย

---

### agent

จัดการเอเจนต์สำหรับ CodePlane

```bash
codeplane agent [command]
```

---

### attach

เชื่อมต่อกับเซิร์ฟเวอร์แบ็กเอนด์ CodePlane ที่กำลังทำงานอยู่ ซึ่งเริ่มต้นผ่านคำสั่ง `serve` หรือ `web`

```bash
codeplane attach [url]
```

วิธีนี้ช่วยให้คุณใช้ TUI กับแบ็กเอนด์ CodePlane ระยะไกลได้ ตัวอย่าง:

```bash
# Start the backend server for web/mobile access
codeplane web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
codeplane attach http://10.20.30.40:4096
```

#### แฟล็ก

| แฟล็ก       | สั้น | คำอธิบาย                               |
| ----------- | ---- | -------------------------------------- |
| `--dir`     |      | ไดเร็กทอรีการทำงานเพื่อเริ่มต้น TUI ใน |
| `--session` | `-s` | ID เซสชันเพื่อดำเนินการต่อ             |

#### create

สร้างเอเจนต์ใหม่

```bash
codeplane agent create
```

คำสั่งนี้จะแนะนำคุณตลอดกระบวนการสร้างเอเจนต์ใหม่ พร้อมพรอมต์ระบบและความสามารถของเครื่องมือ

---

#### list

แสดงรายชื่อเอเจนต์ที่พร้อมใช้งานทั้งหมด

```bash
codeplane agent list
```

### auth

จัดการการตรวจสอบสิทธิ์และข้อมูลรับรอง

```bash
codeplane auth [command]
```

---

#### login

CodePlane ได้รับการกำหนดค่าไว้ล่วงหน้าด้วยรายชื่อผู้ให้บริการทั่วไปที่ [Models.dev](https://models.dev) ดังนั้นคุณเพียงแค่ต้องใส่คีย์ API สำหรับผู้ให้บริการที่คุณต้องการใช้ ข้อมูลนี้จะถูกเก็บไว้ที่ `~/.local/share/codeplane/auth.json`

```bash
codeplane auth login
```

เมื่อ CodePlane เริ่มต้น ระบบจะโหลดข้อมูลรับรองจากไฟล์นี้ ดังนั้นคุณจึงไม่ต้องกังวลกับการตั้งค่าตัวแปรสภาพแวดล้อมหรือไฟล์ `.env` ในโปรเจ็กต์ของคุณ

---

#### list

แสดงรายการข้อมูลรับรองการตรวจสอบสิทธิ์ทั้งหมดที่กำหนดค่าไว้

```bash
codeplane auth list
```

หรือใช้นามแฝงสั้น

```bash
codeplane auth ls
```

---

#### logout

ออกจากระบบของผู้ให้บริการและลบข้อมูลรับรองออกจากไฟล์ auth

```bash
codeplane auth logout
```

### github

จัดการเอเจนต์ GitHub สำหรับเวิร์กโฟลว์อัตโนมัติ

```bash
codeplane github [command]
```

---

#### install

ติดตั้งเอเจนต์ GitHub ใน repository ของคุณ

```bash
codeplane github install
```

คำสั่งนี้จะสร้างเวิร์กโฟลว์ GitHub Actions และแนะนำคุณตลอดกระบวนการตั้งค่า [เรียนรู้เพิ่มเติม](/docs/github)

---

#### run

รันเอเจนต์ GitHub ด้วยตนเองหรือใน GitHub Actions

```bash
codeplane github run
```

##### แฟล็ก

| แฟล็ก     | คำอธิบาย                               |
| --------- | -------------------------------------- |
| `--event` | เหตุการณ์ GitHub เพื่อทริกเกอร์เอเจนต์ |
| `--token` | GitHub token                           |

### mcp

จัดการเซิร์ฟเวอร์ Model Context Protocol (MCP)

```bash
codeplane mcp [command]
```

---

#### add

เพิ่มเซิร์ฟเวอร์ MCP ลงในการกำหนดค่าของคุณ

```bash
codeplane mcp add
```

คำสั่งนี้จะแนะนำคุณผ่านวิซาร์ดเพื่อกำหนดค่าเซิร์ฟเวอร์ MCP ใหม่

---

#### list

แสดงรายชื่อเซิร์ฟเวอร์ MCP ทั้งหมดที่กำหนดค่าไว้และสถานะ

```bash
codeplane mcp list
```

หรือใช้นามแฝงสั้น

```bash
codeplane mcp ls
```

---

#### auth

ยืนยันตัวตนกับเซิร์ฟเวอร์ MCP ที่ต้องการ OAuth

```bash
codeplane mcp auth [name]
```

หากไม่ระบุชื่อเซิร์ฟเวอร์ คุณจะได้รับแจ้งให้เลือกจากเซิร์ฟเวอร์ที่รองรับ OAuth ที่พร้อมใช้งาน

คุณยังสามารถแสดงรายการสถานะการตรวจสอบสิทธิ์ OAuth ได้:

```bash
codeplane mcp auth list
```

หรือใช้นามแฝงสั้น

```bash
codeplane mcp auth ls
```

---

#### logout

ลบข้อมูลรับรอง OAuth สำหรับเซิร์ฟเวอร์ MCP

```bash
codeplane mcp logout [name]
```

#### debug

ดีบักการเชื่อมต่อเซิร์ฟเวอร์ MCP

```bash
codeplane mcp debug <name>
```

### models

แสดงรายการโมเดลที่พร้อมใช้งานทั้งหมดจากผู้ให้บริการที่กำหนดค่าไว้

```bash
codeplane models [provider]
```

คำสั่งนี้จะแสดง ID โมเดลทั้งหมดในรูปแบบ `provider/model`

คุณสามารถใช้ชื่อโมเดลเหล่านี้ในการกำหนดค่า [config](/docs/config/) ของคุณ

คุณยังสามารถส่งผ่านรหัสผู้ให้บริการเพื่อกรองโมเดลตามผู้ให้บริการรายนั้นได้

```bash
codeplane models anthropic
```

#### แฟล็ก

| แฟล็ก       | คำอธิบาย                                               |
| ----------- | ------------------------------------------------------ |
| `--refresh` | รีเฟรชแคชโมเดลจาก models.dev                           |
| `--verbose` | แสดงรายละเอียดโมเดลเพิ่มเติม (รวมข้อมูลเมตาเช่นต้นทุน) |
| `--json`    | แสดงผลลัพธ์เป็น JSON                                   |

ใช้แฟล็ก `--refresh` เพื่ออัปเดตรายการโมเดลที่แคชไว้ มีประโยชน์เมื่อมีการเพิ่มโมเดลใหม่ให้กับผู้ให้บริการและคุณต้องการเห็นใน CodePlane

```bash
codeplane models --refresh
```

### run

รัน CodePlane โดยไม่ต้องใช้ TUI โดยส่งพรอมต์โดยตรง

```bash
codeplane run [message..]
```

มีประโยชน์สำหรับการทำงานอัตโนมัติหรือเมื่อต้องการคำตอบอย่างรวดเร็วโดยไม่ต้องเปิด TUI ตัวอย่าง:

```bash "codeplane run"
codeplane run Explain the use of context in Go
```

คุณยังสามารถแนบไปกับเซิร์ฟเวอร์ `codeplane serve` ที่ทำงานอยู่ หรือใช้เซิร์ฟเวอร์ MCP แบบ headless:

```bash
# Start a headless server in one terminal
codeplane serve

# In another terminal, run commands that attach to it
codeplane run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### แฟล็ก

| แฟล็ก        | สั้น | คำอธิบาย                                                               |
| ------------ | ---- | ---------------------------------------------------------------------- |
| `--command`  |      | คำสั่งที่จะรัน (ใช้ส่วนที่เหลือของ args เป็นอาร์กิวเมนต์)              |
| `--continue` | `-c` | ดำเนินการต่อจากเซสชันล่าสุด                                            |
| `--session`  | `-s` | ID เซสชันเพื่อดำเนินการต่อ                                             |
| `--fork`     |      | แยกเซสชันเมื่อทำต่อ (ใช้กับ `--continue` หรือ `--session`)             |
| `--share`    |      | สร้างลิงก์แชร์สำหรับเซสชัน                                             |
| `--model`    | `-m` | โมเดลที่จะใช้ในรูปแบบ provider/model                                   |
| `--agent`    |      | เอเจนต์ที่จะใช้                                                        |
| `--file`     | `-f` | แนบไฟล์ไปกับข้อความ                                                    |
| `--format`   |      | รูปแบบเอาต์พุต: text (จัดรูปแบบ) หรือ json (JSON ดิบ)                  |
| `--title`    |      | ชื่อสำหรับเซสชัน (หากไม่ได้ระบุ จะสร้างจากพรอมต์)                      |
| `--attach`   |      | แนบไปกับเซิร์ฟเวอร์ codeplane ที่ทำงานอยู่ (เช่น http://localhost:4096) |
| `--port`     |      | พอร์ตสำหรับเซิร์ฟเวอร์ภายในเครื่อง (หากไม่ได้ระบุ จะใช้พอร์ตสุ่ม)      |

### serve

เริ่มต้นเซิร์ฟเวอร์ CodePlane headless ที่ให้บริการ API HTTP ดู [เอกสารเซิร์ฟเวอร์](/docs/server) สำหรับรายละเอียด API เต็มรูปแบบ

```bash
codeplane serve
```

คำสั่งนี้จะเริ่มต้นเซิร์ฟเวอร์ HTTP ให้ API เข้าถึงฟังก์ชันการทำงานของ CodePlane ได้โดยไม่ต้องมี TUI นอกจากนี้ยังรองรับการตรวจสอบสิทธิ์พื้นฐาน HTTP (ชื่อผู้ใช้เริ่มต้นคือ `codeplane` และรหัสผ่านระบุโดย `CODEPLANE_SERVER_PASSWORD`)

#### แฟล็ก

| แฟล็ก        | คำอธิบาย                               |
| ------------ | -------------------------------------- |
| `--port`     | พอร์ตที่จะฟัง                          |
| `--hostname` | ชื่อโฮสต์ที่จะฟัง                      |
| `--mdns`     | เปิดใช้งานการค้นหา mDNS                |
| `--cors`     | ต้นกำเนิดเพิ่มเติมที่อนุญาตสำหรับ CORS |

### stats

แสดงสถิติการใช้งานและค่าใช้จ่ายสำหรับ CodePlane ของคุณ


| แฟล็ก       | คำอธิบาย                                             |
| ----------- | ---------------------------------------------------- |
| `--days`    | แสดงสถิติของ N วันที่ผ่านมา (ค่าเริ่มต้น: ตลอดเวลา)  |

### uninstall

ถอนการติดตั้ง CodePlane และลบข้อมูลการกำหนดค่าทั้งหมด

```bash
codeplane uninstall
```

#### แฟล็ก

| แฟล็ก           | สั้น | คำอธิบาย                            |
| --------------- | ---- | ----------------------------------- |
| `--keep-config` | `-c` | เก็บไฟล์การกำหนดค่าไว้              |
| `--keep-data`   | `-d` | เก็บไฟล์ข้อมูล (เซสชันและสแน็ปช็อต) |
| `--dry-run`     |      | แสดงสิ่งที่จะลบออกโดยไม่ต้องทำจริง  |
| `--force`       | `-f` | บังคับลบโดยไม่มีการแจ้งเตือน        |

### upgrade

อัปเกรด CodePlane เป็นเวอร์ชันล่าสุดหรือเวอร์ชันที่ระบุ

```bash
codeplane upgrade [target]
```

อัปเกรดเป็นเวอร์ชันล่าสุด

```bash
codeplane upgrade
```

อัปเกรดเป็นเวอร์ชันที่ระบุ

```bash
codeplane upgrade v0.1.48
```

#### แฟล็ก

| แฟล็ก      | สั้น | คำอธิบาย                                        |
| ---------- | ---- | ----------------------------------------------- |
| `--method` | `-m` | วิธีการติดตั้งที่ใช้ curl, npm, pnpm, bun, brew |

## แฟล็กระดับโลก

CodePlane CLI ยอมรับแฟล็กสากลต่อไปนี้สำหรับทุกคำสั่ง

| แฟล็ก          | สั้น | คำอธิบาย                                  |
| -------------- | ---- | ----------------------------------------- |
| `--help`       | `-h` | แสดงความช่วยเหลือ                         |
| `--version`    | `-v` | พิมพ์เวอร์ชัน                             |
| `--print-logs` |      | พิมพ์บันทึกไปยัง stderr                   |
| `--log-level`  |      | ระดับการบันทึก (DEBUG, INFO, WARN, ERROR) |

## ตัวแปรสภาพแวดล้อม

CodePlane สามารถกำหนดค่าโดยใช้ตัวแปรสภาพแวดล้อม

| ตัวแปร                                | ชนิด    | คำอธิบาย                                                         |
| ------------------------------------- | ------- | ---------------------------------------------------------------- |
| `CODEPLANE_AUTO_SHARE`                 | Boolean | แชร์เซสชันอัตโนมัติเมื่อสร้าง                                    |
