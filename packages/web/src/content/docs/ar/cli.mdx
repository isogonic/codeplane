The Codeplane CLI starts the web app when run without any arguments.

```bash
codeplane
```

But it also accepts commands as documented on this page. This allows you to interact with Codeplane programmatically.

```bash
codeplane run "Explain how closures work in JavaScript"
```

---

## Commands

The Codeplane CLI also has the following commands.

---

### agent

Manage agents for Codeplane.

```bash
codeplane agent [command]
```

#### create

Create a new agent with custom configuration.

```bash
codeplane agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
codeplane agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
codeplane auth [command]
```

---

#### login

Codeplane is powered by the provider list at [Models.dev](https://models.dev), so you can use `codeplane auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/codeplane/auth.json`.

```bash
codeplane auth login
```

When Codeplane starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
codeplane auth list
```

Or the short version.

```bash
codeplane auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
codeplane auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
codeplane github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
codeplane github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
codeplane github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

---

### mcp

Manage Model Context Protocol servers.

```bash
codeplane mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
codeplane mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
codeplane mcp list
```

Or use the short version.

```bash
codeplane mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
codeplane mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
codeplane mcp auth list
```

Or use the short version.

```bash
codeplane mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
codeplane mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
codeplane mcp debug <name>
```

### models

List all available models from configured providers.

```bash
codeplane models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
codeplane models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Codeplane.

```bash
codeplane models --refresh
```

### run

Run codeplane in non-interactive mode by passing a prompt directly.

```bash
codeplane run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without opening the web app. For example.

```bash "codeplane run"
codeplane run Explain the use of context in Go
```

You can also attach to a running `codeplane serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
codeplane serve

# In another terminal, run commands that attach to it
codeplane run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag                             | Short | Description                                                             |
| -------------------------------- | ----- | ----------------------------------------------------------------------- |
| `--command`                      |       | The command to run, use message for args                                |
| `--continue`                     | `-c`  | Continue the last session                                               |
| `--session`                      | `-s`  | Session ID to continue                                                  |
| `--fork`                         |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--share`                        |       | Share the session                                                       |
| `--model`                        | `-m`  | Model to use in the form of provider/model                              |
| `--agent`                        |       | Agent to use                                                            |
| `--file`                         | `-f`  | File(s) to attach to message                                            |
| `--format`                       |       | Format: default (formatted) or json (raw JSON events)                   |
| `--title`                        |       | Title for the session (uses truncated prompt if no value provided)      |
| `--attach`                       |       | Attach to a running codeplane server (e.g., http://localhost:4096)       |
| `--port`                         |       | Port for the local server (defaults to random port)                     |
| `--dangerously-skip-permissions` |       | Auto-approve permissions that are not explicitly denied (dangerous!)    |

### serve

Start a headless Codeplane server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
codeplane serve
```

This starts an HTTP server that provides API access to codeplane functionality without opening the browser UI. Set `CODEPLANE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `codeplane`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

### session

Manage Codeplane sessions.

```bash
codeplane session [command]
```

---

#### list

List all Codeplane sessions.

```bash
codeplane session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your Codeplane sessions.

```bash
codeplane stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
codeplane export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Codeplane share URL.

```bash
codeplane import <file>
```

You can import from a local file or an Codeplane share URL.

```bash
codeplane import session.json
codeplane import https://opncd.ai/s/abc123
```

---

### web

Start a headless Codeplane server with a web interface.

```bash
codeplane web
```

This starts an HTTP server and opens a web browser to access Codeplane through a web interface. Set `CODEPLANE_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `codeplane`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
codeplane acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Codeplane and remove all related files.

```bash
codeplane uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates codeplane to the latest version or a specific version.

```bash
codeplane upgrade [target]
```

To upgrade to the latest version.

```bash
codeplane upgrade
```

To upgrade to a specific version.

```bash
codeplane upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Codeplane can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `CODEPLANE_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `CODEPLANE_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `CODEPLANE_CONFIG`                     | string  | Path to config file                               |
| `CODEPLANE_CONFIG_DIR`                 | string  | Path to config directory                          |
| `CODEPLANE_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `CODEPLANE_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `CODEPLANE_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `CODEPLANE_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `CODEPLANE_PERMISSION`                 | string  | Inlined json permissions config                   |
| `CODEPLANE_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `CODEPLANE_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `CODEPLANE_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `CODEPLANE_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `CODEPLANE_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `CODEPLANE_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `CODEPLANE_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `CODEPLANE_DISABLE_MODELS_FETCH`       | boolean | Disable fetching models from remote sources       |
| `CODEPLANE_FAKE_VCS`                   | string  | Fake VCS provider for testing purposes            |
| `CODEPLANE_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `CODEPLANE_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `CODEPLANE_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `CODEPLANE_SERVER_USERNAME`            | string  | Override basic auth username (default `codeplane`) |
| `CODEPLANE_MODELS_URL`                 | string  | Custom URL for fetching models configuration      |
