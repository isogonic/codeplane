Desktop

Native macOS, Windows, and Linux app — same UI as /web, wrapped in an Electron shell so it gets a dock icon, native window chrome, and auto-updates without the browser.

Install

Direct downloads:

  • macOS Apple Silicon (.dmg)
  • macOS Intel (.dmg)
  • Windows x64 (.exe)
  • Linux x64 (.AppImage)

Full install matrix at Install.

Connecting to a server

The desktop app is a client — it doesn't run the agent locally. On first launch you'll see a server picker. Three paths:

  1. Local. Hit Add local server; the desktop app spawns a Codeplane server in the background and connects on http://localhost:4096.
  2. Remote. Paste the URL of a Codeplane server you run on a VPS / homelab (see Self-hosting).
  3. Saved instance. Servers you've added via codeplane instance add show up here automatically.

Updates

The desktop app updates itself silently via electron-updater. On launch it checks GitHub Releases for a newer latest-mac.yml / latest.yml / latest-linux.yml, downloads the delta, and applies it the next time you quit + reopen. You don't need to revisit the download page after the first install.

To pin a version, install from the .dmg / .exe / .AppImage of that release directly. The auto-updater respects skip-version requests for one cycle.

Native integrations

  • System tray / menu bar — running sessions show a status dot. Click to surface the window.
  • Global shortcut — ⌥⌘C on macOS toggles the window from anywhere.
  • File-open intent — drop a folder onto the dock icon to open it as a project.
  • Notifications — system-level alerts when a long-running session finishes.

What's different from the web UI

  • Multi-server picker (web only knows about the server it was opened from).
  • Local-server boot button — no need to codeplane web from a shell first.
  • Drag-drop a project folder onto the dock to register it.
  • Auto-update; the browser tab needs a manual reload after a server upgrade.
