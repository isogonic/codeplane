Instances

A Codeplane instance is a server endpoint your client knows about. Manage many — local laptop, VPS, teammate's machine — from one address book and switch per-session.

List what you have

codeplane instance list

Prints a table of name, URL, last-connected, default-marker. The CLI, web picker, desktop server-switcher, and mobile shell all read from the same list.

Add an instance

codeplane instance add prod https://codeplane.example.com
codeplane instance add laptop http://192.168.1.42:4096 --auth eyJ...

The --auth flag stores a Bearer token alongside the URL. Subsequent connections send it automatically.

Pick a default

codeplane instance default prod

The default instance is what codeplane tui / codeplane web connect to when no --instance flag is passed.

Remove one

codeplane instance remove laptop

Shared local runtime

Every front-end can boot a managed local server on demand — that's the "Local server" entry you see in the desktop / mobile picker. The runtime is shared across instances; codeplane instance runtime --reset wipes the cached version.

Where the address book lives

$CODEPLANE_HOME/instances.json (default ~/.codeplane/instances.json). Plain JSON; safe to edit by hand or sync between machines with a dotfile manager.
