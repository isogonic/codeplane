CLI reference

The codeplane binary is the single entry point. Every front-end (TUI, web, desktop, mobile) ultimately routes through the same server it boots. The commands below cover everything you can do from a shell.

Synopsis

codeplane <command> [options]

Commands:
  completion              generate shell completion script
  serve                   start a headless codeplane server
  web                     start a server and open the web interface
  tui                     start the terminal UI
  instance                manage saved Codeplane instances + shared runtime
  upgrade [target]        upgrade codeplane to the latest (or specific) version

Options:
  -h, --help              show help
  -v, --version           show version number
      --print-logs        print logs to stderr
      --log-level <lvl>   DEBUG, INFO, WARN, or ERROR
      --pure              run without external plugins

codeplane serve

Boots the HTTP + WebSocket server with no UI. Every other Codeplane client (desktop, mobile, web) eventually talks to this.

codeplane serve [options]

  --port <n>          port to bind (default: 4096)
  --hostname <host>   bind address (default: 127.0.0.1; use 0.0.0.0 for LAN)
  --share             print a connection QR + share URL for the mobile app
  --instance <name>   run under a named instance (see `codeplane instance`)
  --auth <token>      require Bearer token on every request
  --no-tls            disable HTTPS even if certs exist (dev / loopback only)

The server is single-process. Killing it stops every connected client gracefully.

codeplane web

Boots the same server + opens the web UI in your default browser. Convenience wrapper around codeplane serve + open.

codeplane web [options]

  --port <n>          port to bind (default: 4096)
  --no-open           start the server but don't auto-launch the browser
  <all serve flags>   every flag from `codeplane serve` is accepted too

codeplane tui

Full-screen terminal UI. Spawns a server in the background if one isn't running.

codeplane tui [options]

  --instance <name>   pick a saved instance (instead of localhost)
  --session <id>      jump straight into an existing session
  --mode <mode>       start in a specific mode (chat, build, plan, ...)

Hit ? inside the TUI for the full keybinding overlay. Quit with q or Ctrl+C.

codeplane instance

Codeplane can talk to multiple servers — your laptop's local one, a remote VPS, a teammate's machine. The instance command manages the address book.

codeplane instance <subcommand>

  list                    list every saved instance
  add <name> <url>        register a remote server
  remove <name>           drop one
  default <name>          set the default for `tui` / `web` etc.
  runtime                 manage the shared local runtime cache

Full guide at Instances.

codeplane upgrade

Re-runs the installer in place. Picks up the latest stable release and replaces the binary.

codeplane upgrade [target]

  target              optional version to pin (e.g. `28.1.20`); defaults to latest
  --beta              follow the beta channel instead of stable
  --check             print the latest available version, don't install
  --force             reinstall even if you're already on the target

codeplane completion

Prints a shell completion script to stdout. Pipe it into the right place for your shell:

# bash
codeplane completion bash > /etc/bash_completion.d/codeplane

# zsh — add to your fpath
codeplane completion zsh > "${fpath[1]}/_codeplane"

# fish
codeplane completion fish > ~/.config/fish/completions/codeplane.fish

Global options

FlagWhat it does
-h, --helpPrint help for the current command + exit.
-v, --versionPrint the semver (e.g. 28.1.20) and exit.
--print-logsEcho internal logs to stderr in real time. Useful when something hangs.
--log-level <DEBUG|INFO|WARN|ERROR>Filter log verbosity. DEBUG shows every request the agent issues.
--pureSkip loading user plugins and project-local plugin code. Handy for debugging.

Environment variables

The CLI reads these (in addition to anything codeplane.json sets).

VariablePurpose
ANTHROPIC_API_KEYDefault key for Anthropic models.
OPENAI_API_KEYDefault key for OpenAI models.
OPENAI_BASE_URLPoint the OpenAI client at an alternative endpoint (Ollama, vLLM, LM Studio).
OPENROUTER_API_KEYDefault key for OpenRouter.
CODEPLANE_HOMEOverride the user data directory (default ~/.codeplane).
CODEPLANE_LOG_LEVELEquivalent to --log-level as an env var.

Exit codes

CodeMeaning
0Success.
1Generic error (bad flag, unreachable instance, etc.). Detail is on stderr.
2Usage error — invalid subcommand or required argument missing.
3Authentication failed (server requires a token you didn't provide).
130You hit Ctrl+C. Sessions are saved before exit.
