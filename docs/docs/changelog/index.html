Changelog

Notable changes per release. Every entry maps to a tag at github.com/devinoldenburg/codeplane/releases where the source binaries + full git log live.

28.2.x

v28.2.2

  • Release infrastructure: full GitLab CI port of the three GitHub release workflows.
  • Site: codeplane.cc ported to a monochrome OpenAI-style docs site with HugeIcons.

v28.1.24

  • Fix: Settings → General Auto-accept permissions toggle now actually toggles. Previously it was bound to a per-directory rule + silently disabled on the global settings page.
  • New global auto-accept flag short-circuits every permission check when on; per-directory + per-session rules still apply when it's off.

v28.1.22

  • Fix: Queued follow-up drag was locked to horizontal — flipped the axis constraint so up/down reorder actually moves the row.

v28.1.21

  • Toast restyle: first action now reads as a filled white CTA against the always-dark toast surface, secondary actions stay as quiet ghost text. Matches shadcn's sonner pattern.

v28.1.12

  • UI refactor: Logic radix-nova design language ported across the shared SolidJS UI via a new shadcn.css token layer.
  • Real @hugeicons/core-free-icons glyphs replace the hand-rolled SVG path map in icon.tsx.
  • Strict light/dark: the 35-theme picker (Dracula, Tokyonight, Catppuccin, etc.) is gone; .dark class on <html> flips the palette via static CSS.
  • Stronger contrast tokens: off-white canvas + pure-white cards in light; deep canvas + clearly-lifted cards in dark.
  • Composer: two rounded cards (shell + tray) with a 4px gap; symmetric py-2 tray padding.
  • Project avatars: deterministic FNV-1a hash assigns each project a distinct colour from the 6-slot palette.
  • ErrorBoundary auto-reloads on Vite stale-chunk imports so the dead-end error screen never shows after an HMR update.
  • New .gitlab-ci.yml + GITLAB_CI.md — Linux-only port of the three GitHub release workflows.

28.0.x

Pre-design-language history: every v28.0.x tag (currently 16 releases) lives at the GitHub releases page with full notes. Highlights:

  • iOS Live Activities for long-running sessions.
  • Multi-instance address book + remote server picker.
  • Queued follow-ups with live reorder.
  • Session revert / branch / share.
  • Plugin SDK (@codeplane-ai/plugin) general availability.

Per-tag notes

Every release ships three tags — base, -desktop, -mobile — each with platform-specific notes attached to its release page on GitHub. Browse them all at /releases.
