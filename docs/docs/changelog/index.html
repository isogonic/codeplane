v28.2.1May 18, 2026

Hotfix

Codeplane v28.2.1 fixes session history opening, session switching, and sidebar load behavior in the app.

Root cause

The app was doing too much session work during bootstrap and sidebar rendering: duplicate full session loads, eager load-all effects, and row-level workspace access that could bootstrap inactive workspaces. During direct session opens, the message panel also had no explicit loading fallback, so session state could briefly look blank while messages hydrated.

Fix

1. packages/app/src/context/global-sync/bootstrap.ts: load the first session page once instead of forcing duplicate full-session loads. 2. packages/app/src/pages/layout/sidebar-workspace.tsx and related sidebar helpers: avoid row-level bootstrapping, index child sessions once per store, and make full-session loading explicit. 3. packages/app/src/pages/layout/sidebar-session-preview.ts: compute session previews without sorting every message list. 4. packages/app/src/pages/session.tsx: show a real loading state while session messages hydrate. 5. packages/shared/src/local-runtime.ts: tighten preferred-version parsing while preserving stable semver build metadata. 6. Release metadata: bump synchronized package versions to 28.2.1.
