Permissions

Codeplane sandboxes every side-effecting action. The first time the agent wants to edit a file, run a shell command, or call a tool it hasn't used before, you get a prompt. Permissions are decided per directory + per session, with optional global override.

Decision categories

CategoryWhat it covers
editAny file write — create, modify, delete.
shellShell commands run via the bash tool.
readReading files (default: allow — turn down for sensitive paths).
netHTTP requests from tools that need the network.
toolAny tool not in the categories above (plugin tools, MCP tools).

Decision values

  • "allow" — go ahead, don't prompt.
  • "ask" — prompt every time. Default for edit + shell.
  • "deny" — refuse silently; the tool call fails.

Rule grammar

"permission": {
  "edit":  "ask",
  "shell": "ask",
  "rules": {
    "shell": {
      "git diff*":      "allow",
      "git log*":       "allow",
      "npm test":       "allow",
      "rm -rf*":        "ask"
    },
    "edit": {
      "**/*.lock":      "deny",
      "node_modules/**":"deny"
    },
    "tool": {
      "mcp__github__*": "ask",
      "mcp__github__create_issue": "allow"
    }
  }
}

Shell globs match the full command line (including args). Edit globs match the file path relative to the project root. Tool patterns match the tool name (mcp__server__tool or plugin tool IDs).

Scopes

An Always grant is stored at one of three scopes, picked at prompt time:

  • Session — applies until the session ends; lost on next session.
  • Directory — applies to every session opened from this project root.
  • Global — applies to every session, every project (see below).

Global auto-accept

Settings → General → Berechtigungen automatisch akzeptieren / Auto-accept permissions flips a global flag that short-circuits every permission check. Useful for trusted local-only setups; never recommended for self-hosted servers exposed to others.

Where decisions live

Persisted in the server's permission store (SQLite, alongside sessions). Clear all auto-accept grants with the Reset auto-accept rules button in Settings → Reset, or by removing the corresponding keys from the store directly.
