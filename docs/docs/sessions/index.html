Sessions

A session is a single thread of agent + user messages, scoped to a project. Sessions persist forever — no auto-deletion, no rolling window.

Lifecycle

  1. Create — click New in any front-end, or hit ⌘N.
  2. Active — busy when the agent is streaming, idle when waiting for input.
  3. Archive — move out of the main sidebar without deleting. Restore from Archived.
  4. Share — generate a public read-only URL signed by the server.

Branching

Hit Branch from here on any assistant message to start a new thread that inherits the timeline up to that point. Useful for exploring alternative reasoning paths without losing the original.

Queued follow-ups

While the agent is busy, additional messages queue up. They render under the active session with a Queued badge; drag to reorder, click Send now to interrupt the current turn with a specific follow-up. Queue order is per-session, server-side, multi-client.

Revert

Hit the Revert to here arrow on a user message to roll the timeline back to that point — every subsequent message, every tool call, every file edit is undone. Files restore from the server's working-tree snapshot the next message took.

Memory

Each session has a free-form memory blob the agent can append to. Use it for "remember the user prefers TypeScript with strict mode" style notes. Project-wide memory lives in .codeplane/memory.md.

Storage

SQLite at $CODEPLANE_HOME/sessions.db. Every message, every tool call, every permission decision lives there. Backup with sqlite3 ... ".backup"; see Self-hosting → Backups.
