Plugins

Write your own tools, agents, or prompt presets that load into every Codeplane session. Plugins are plain TypeScript modules registered via @codeplane-ai/plugin.

What a plugin can do

  • Add tools — first-class entries in the agent's tool registry, with schema, permission flow, streaming output.
  • Add agents — named personas with their own prompt, model, and tool overrides.
  • Add prompts — reusable slash-command prompts that show up in / autocomplete.
  • Hook events — session lifecycle, message stream, permission grant.

For tools that already exist as MCP servers, prefer MCP — plugins are for the cases where you want tighter integration (UI widgets, event hooks, streaming output).

Hello-world plugin

import { definePlugin, tool } from "@codeplane-ai/plugin"

export default definePlugin({
  id: "my-org.weather",
  version: "0.1.0",
  tools: [
    tool({
      name: "weather",
      description: "Look up the current weather for a city.",
      input: { city: { type: "string", required: true } },
      async run({ input }) {
        const res = await fetch(`https://wttr.in/${input.city}?format=3`)
        return { content: await res.text() }
      },
    }),
  ],
})

Loading a plugin

Drop the file in ~/.codeplane/plugins/weather.ts (user-wide) or <project>/.codeplane/plugins/weather.ts (per-project). Codeplane runs each plugin in an isolated Bun-managed worker on session start.

Or import from npm:

// codeplane.json
{
  "plugins": ["@my-org/codeplane-weather"]
}

Permission model

Plugin tools land in the same permission flow as everything else — the user sees an Allow / Deny / Always prompt on first call, persisted under the session or directory scope. Tools that touch the filesystem or shell inherit the standard edit / shell approval rules.

The full API

The plugin SDK ships type-complete: definePlugin, tool, agent, prompt, onSession, onMessage, onPermission. Full reference at TypeScript SDK. Source + examples live at packages/plugin.

Distributing

Publish as a regular npm package; users add the name to "plugins" in their codeplane.json. Versions are pinned via npm semver. Codeplane verifies plugin signatures when shipped via codeplane.lock (opt-in).
