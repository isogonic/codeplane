Web

The Codeplane web UI runs in any modern browser. It's the canonical UI — the desktop + mobile shells both wrap it.

Open it

codeplane web                # boot a local server + open the browser
codeplane serve --port 4096  # server only — visit http://localhost:4096 by hand

The web UI is a SolidJS SPA served by the Codeplane server. Once loaded, it talks back via HTTP + Server-Sent-Events; no client-side build, no node_modules to manage.

What's in the UI

Five regions, all keyboard-accessible:

  • Top bar — server picker, status, settings, quick-switcher (⌘K).
  • Left rail — projects + sessions tree.
  • Center pane — message timeline. Streamed thinking, tool calls, diffs, terminal output, browser previews.
  • Right rail — review tab (working tree diff), timeline (file changes), terminal pane, todos panel.
  • Composer — bottom dock. Mode + model + agent pickers in the tray below.

Sharing a session

Each session has a stable URL. Copy it from the browser bar to share — anyone with access to your Codeplane server can open it. Sessions can also be exported as Markdown via the toolbar's Share button (works without a server account, signed snapshot).

Themes

Light, dark, system — via Settings → Appearance → Color scheme. The palette is the Logic radix-nova monochrome — see Themes for the token reference.

Keybindings

All listed at Keybinds. The most useful:

KeyAction
⌘KQuick switcher — sessions, projects, files, commands.
⌘NNew session.
⌘BToggle left sidebar.
⌘/Search the open session.
⌘,Settings dialog.

Browser support

Tested on the latest two stable versions of Chrome / Edge, Firefox, and Safari. Requires support for: ES2022, Server-Sent Events, OKLCH colour, :has() selector. Internet Explorer, ancient Edge, and headless browsers without CSS-color-4 won't render correctly.
