Configuration

Codeplane reads codeplane.json from three locations, in order — every file merges over the previous one, so project settings beat user settings, which beat the bundled defaults.

Where it lives

LayerPathUse it for
Defaultsshipped inside the binaryprovider templates, MCP server entries, agent presets
User~/.codeplane/codeplane.jsonAPI keys, preferred model, theme
Project<project>/.codeplane/codeplane.jsonper-repo rules, permission overrides, project agents

The schema is published at https://codeplane.cc/config.json — point your editor at it for autocomplete:

{
  "$schema": "https://codeplane.cc/config.json",
  ...
}

Top-level shape

{
  "$schema": "https://codeplane.cc/config.json",
  "provider":   { /* models + auth */ },
  "permission": { /* what the agent may do without asking */ },
  "agent":      { /* personalities the user can switch between */ },
  "mode":       { /* per-mode prompt + tool overrides */ },
  "mcp":        { /* Model Context Protocol servers */ },
  "rules":      { /* per-directory rule files */ },
  "commit":     { /* git commit author / co-author config */ },
  "telemetry":  false,
  "theme":      "system"
}

provider

Pick the LLM provider(s) Codeplane talks to. Multiple are fine — sessions can switch on the fly.

"provider": {
  "anthropic": {
    "apiKey": "{env:ANTHROPIC_API_KEY}",
    "default": "claude-sonnet-4-6"
  },
  "openai": {
    "apiKey": "{env:OPENAI_API_KEY}",
    "baseUrl": "{env:OPENAI_BASE_URL}",
    "default": "gpt-5.2"
  },
  "openrouter": {
    "apiKey": "{env:OPENROUTER_API_KEY}"
  },
  "ollama": {
    "baseUrl": "http://localhost:11434/v1"
  }
}

String values may use {env:VAR} placeholders that resolve at load time. default is optional — falls back to the first non-empty provider's default model.

permission

Codeplane sandboxes the agent's actions. Every edit, shell, or read outside a marked-safe path prompts you. Tune the gates here:

"permission": {
  "edit":  "ask",
  "shell": "ask",
  "read":  "allow",
  "rules": {
    "shell": {
      "npm:*":      "allow",
      "git diff*":  "allow",
      "rm -rf*":    "ask"
    },
    "edit": {
      "**/*.lock":  "deny"
    }
  }
}

Values per category: "allow" (silent), "ask" (prompt every time), "deny" (refuse). Globs inside rules.shell match the full command line; rules.edit globs match the target path. See Permissions for the full grammar.

agent

An agent is a named persona — a system prompt plus tool/model overrides. The user picks one in the composer; you can ship a project-specific set.

"agent": {
  "build": {
    "description": "Implement features, edit files, run tests.",
    "model":  "anthropic:claude-sonnet-4-6",
    "prompt": "You are a careful senior engineer..."
  },
  "review": {
    "description": "Read-only code reviewer.",
    "model":  "anthropic:claude-opus-4-7",
    "tools":  { "edit": "deny", "shell": "deny" }
  }
}

mode

Modes are lightweight: different prompts, sometimes different tools, all on top of the active agent. chat, build, plan, and shell ship by default.

"mode": {
  "plan": {
    "description": "Think first, no file edits.",
    "tools": { "edit": "deny" },
    "prompt": "Produce a step-by-step plan before any action..."
  }
}

mcp

Wire any Model Context Protocol server as a tool source. Codeplane spawns the process at session start and pipes JSON-RPC over stdio.

"mcp": {
  "filesystem": {
    "command": "npx",
    "args":    ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/Code"]
  },
  "github": {
    "command": "uvx",
    "args":    ["mcp-server-github"],
    "env":     { "GITHUB_TOKEN": "{env:GITHUB_TOKEN}" }
  }
}

Full configuration + bundled servers at MCP servers.

rules

Pointers to one or more rule files (Markdown) that get injected into the agent's system prompt. Useful for code-style docs, ADRs, etc.

"rules": [
  "./AGENTS.md",
  "./.codeplane/rules.md"
]

commit

Used when Codeplane creates a git commit on your behalf (the agent calls the git tool).

"commit": {
  "coauthor": true,
  "author":   { "name": "Devin", "email": "devin@logicplanes.com" }
}

When coauthor is true, commits include a Co-Authored-By: Codeplane <noreply@codeplane.cc> trailer.

theme

Default colour scheme. "system" follows the OS; "light" / "dark" force a specific palette. The desktop and web UIs respect this on first load and let the user override.

"theme": "system"

telemetry

Reporting is disabled by default. Codeplane has no telemetry endpoint — the key exists so you can confirm it's off in audits.

"telemetry": false

Validating

Run any of the front-ends with --log-level DEBUG and Codeplane prints the merged config + schema violations to stderr on first boot. Or jq-pipe through the schema:

jq . codeplane.json && \
  curl -fsSL https://codeplane.cc/config.json | \
  npx ajv-cli validate -s /dev/stdin -d codeplane.json
