HTTP API

The Codeplane server speaks HTTP + Server-Sent Events. Every front-end (CLI, TUI, web, desktop, mobile) is built on top of it. The TypeScript SDK at /docs/sdk wraps these endpoints; this page is for direct callers.

Base URL + auth

Default: http://localhost:4096. When the server was started with --auth <token>, send Authorization: Bearer <token> on every request.

Conventions

  • JSON in, JSON out. Bodies use Content-Type: application/json.
  • IDs are opaque strings; clients never compose them.
  • Errors return HTTP 4xx / 5xx with {"error": {"code": "...", "message": "..."}}.
  • Streaming endpoints use Server-Sent Events — set Accept: text/event-stream on the request.

Projects

MethodPathWhat
GET/v2/projectList all known projects.
POST/v2/projectRegister a worktree as a project. Body: { directory }.
GET/v2/project/:idFetch one.
DELETE/v2/project/:idUnregister (sessions are kept).

Sessions

MethodPathWhat
GET/v2/session?projectID=<id>List sessions in a project.
POST/v2/sessionCreate a session. Body: { projectID, agent?, mode? }.
POST/v2/session/:id/send (SSE)Send a message; stream the agent's reply.
POST/v2/session/:id/abortStop the current turn.
POST/v2/session/:id/archiveArchive (hide from sidebar).
POST/v2/session/:id/shareGenerate a public read-only link.
DELETE/v2/session/:idHard delete + cascade.

Messages

MethodPathWhat
GET/v2/message?sessionID=<id>Replay every message in a session.
GET/v2/message/:idFetch a single message + all its parts.

Permissions

MethodPathWhat
GET/v2/permission?directory=<path>List pending permission requests.
POST/v2/permission/respondBody: { sessionID, permissionID, response: "once"|"always"|"reject" }.

Events (SSE)

Subscribe to every server event in one stream:

GET /v2/event
Accept: text/event-stream

event: session.created
data: { "id": "ses_abc", "projectID": "prj_123", ... }

event: message.part
data: { "sessionID": "ses_abc", "messageID": "msg_def", "delta": "Hello " }

event: permission.asked
data: { "sessionID": "ses_abc", "id": "perm_xyz", "tool": "edit", ... }

The stream stays open until you close it. Reconnect logic should backoff exponentially; the server replays missed events when you send Last-Event-ID.

Config

MethodPathWhat
GET/v2/config?directory=<path>Resolved + merged config for a directory.
PATCH/v2/configUpdate a slice. Body: { directory?, patch }.

OpenAPI spec

A machine-readable OpenAPI document lives at /api/openapi.json on every server (note the leading /api, not /v2). Generate clients with openapi-generator or skim it in Swagger UI.
